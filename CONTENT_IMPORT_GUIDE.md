# Руководство по импорту программ обучения из CSV

Это руководство поможет добавить реальные программы обучения на сайт **без знания программирования**.

Вы заполняете таблицу Excel или Google Таблиц, сохраняете её как CSV и запускаете одну команду. Скрипт создаст файлы в папке `src/content/courses/`.

---

## 1. Где находится таблица

| Файл                               | Назначение                                                          |
| ---------------------------------- | ------------------------------------------------------------------- |
| `data/courses-import-template.csv` | Шаблон с примером строки. **Не удаляйте** этот файл.                |
| `data/courses-import.csv`          | Ваша рабочая таблица для импорта. Создайте её копированием шаблона. |

**Как начать:**

1. Откройте папку `data/` в проекте.
2. Скопируйте файл `courses-import-template.csv`.
3. Переименуйте копию в `courses-import.csv`.
4. Заполняйте `courses-import.csv` — в шаблоне оставьте пример или удалите демонстрационную строку перед реальным импортом.

---

## 2. Чем открыть таблицу

Подойдут:

- **Microsoft Excel**
- **Google Таблицы** (бесплатно, в браузере)
- **LibreOffice Calc** (бесплатная программа)

Рекомендуем Google Таблицы или Excel — они удобно показывают колонки.

---

## 3. Как сохранить CSV в UTF-8

Кодировка **UTF-8** нужна, чтобы русский текст не превратился в «кракозябры».

### Excel (Windows)

1. Файл → Сохранить как.
2. Тип файла: **CSV UTF-8 (разделитель — запятая) (\*.csv)**.
3. Сохраните как `courses-import.csv` в папку `data/`.

### Google Таблицы

1. Файл → Скачать → **Значения, разделённые запятыми (.csv)**.
2. Переместите скачанный файл в папку `data/`.
3. Переименуйте в `courses-import.csv`.

### LibreOffice Calc

1. Файл → Сохранить как.
2. Тип: **Текст CSV**.
3. В параметрах выберите кодировку **UTF-8**.

## 4. Что означает каждая колонка

| Колонка            | Описание                                                                                                                                        |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`               | Уникальный идентификатор программы (латиница, без пробелов). Например: `neyroseti-chatgpt-2026`                                                 |
| `slug`             | Имя файла на сайте. Только латиница, цифры и дефисы. Например: `chatgpt-dlya-nachinayushchih`. Если пусто — создаётся автоматически из названия |
| `title`            | Полное название программы                                                                                                                       |
| `shortTitle`       | Короткое название для карточек (можно оставить пустым — подставится `title`)                                                                    |
| `category`         | Направление: `neyroseti`, `marketing`, `psihologiya`, `zdorove`, `dizayn`, `angliyskiy`                                                         |
| `subcategory`      | Подкатегория, например «SMM» или «ChatGPT»                                                                                                      |
| `school`           | Название школы или платформы                                                                                                                    |
| `author`           | Автор или преподаватель (если есть)                                                                                                             |
| `shortDescription` | Краткое описание для карточки (1–2 предложения)                                                                                                 |
| `fullDescription`  | Полное описание для страницы программы                                                                                                          |
| `type`             | Тип: `course`, `profession`, `skill`, `intensive`, `marathon`                                                                                   |
| `isFree`           | Бесплатная программа: `true` или `false` (также: `да` / `нет`)                                                                                  |
| `hasFreeTrial`     | Есть пробный урок: `true` или `false`                                                                                                           |
| `priceText`        | Текст о стоимости, например «Бесплатно» или «от 4 990 ₽»                                                                                        |
| `level`            | Уровень: `beginner`, `intermediate`, `advanced`                                                                                                 |
| `format`           | Формат: `video`, `live`, `mixed`, `text`, `webinar`, `self-paced`                                                                               |
| `duration`         | Продолжительность, например «4–6 недель»                                                                                                        |
| `audience`         | Для кого программа. Список через `\|`                                                                                                           |
| `learningOutcomes` | Чему научится слушатель. Список через `\|`                                                                                                      |
| `curriculum`       | Содержание / модули. Список через `\|`                                                                                                          |
| `advantages`       | Преимущества. Список через `\|`                                                                                                                 |
| `limitations`      | Ограничения или важные оговорки. Список через `\|`                                                                                              |
| `certificate`      | Выдаётся сертификат: `true` / `false`                                                                                                           |
| `feedback`         | Есть обратная связь: `true` / `false`                                                                                                           |
| `image`            | Путь к изображению (если используете), например `/images/courses/example.jpg`                                                                   |
| `officialUrl`      | Официальная страница программы на сайте школы                                                                                                   |
| `affiliateUrl`     | Ваша партнёрская ссылка (для перехода с сайта)                                                                                                  |
| `partnerProgram`   | Название партнёрской программы, например `infohit`                                                                                              |
| `tags`             | Теги для фильтров. Список через `\|`                                                                                                            |
| `featured`         | Показывать в блоке «избранное» на сайте: `true` / `false`                                                                                       |
| `recommended`      | Рекомендуемая программа: `true` / `false`                                                                                                       |
| `priority`         | Приоритет сортировки (число, чем больше — тем выше)                                                                                             |
| `status`           | Статус: `draft` (черновик), `active` (опубликован), `paused`, `archived`                                                                        |
| `checkedAt`        | Дата проверки данных: `ГГГГ-ММ-ДД`, например `2026-07-09`                                                                                       |
| `publishedAt`      | Дата публикации (необязательно)                                                                                                                 |
| `updatedAt`        | Дата обновления (необязательно)                                                                                                                 |
| `seoTitle`         | SEO-заголовок страницы                                                                                                                          |
| `seoDescription`   | SEO-описание страницы                                                                                                                           |
| `disclaimer`       | Юридическая оговорка под описанием                                                                                                              |

## 5. Обязательные поля

Для любой строки обязательно заполните:

- `id`
- `title`
- `category`
- `shortDescription`
- `seoTitle`
- `seoDescription`
- `checkedAt`

Для публикации со статусом `active` дополнительно нужна партнёрская ссылка `affiliateUrl`. Если она не заполнена, программа **автоматически останется черновиком** (`draft`).

---

## 6. Как записывать списки через `|`

Если в одной ячейке несколько пунктов, разделяйте их символом `|` (вертикальная черта):

```
Начинающие | Те, кто хочет сменить профессию | Фрилансеры
```

Скрипт превратит это в список в файле программы.

**Важно:** если в пункте есть запятая, заключите всю ячейку в кавычки в CSV (Excel и Google Таблицы делают это автоматически при экспорте).

---

## 7. Как указать бесплатный курс

В колонках:

- `isFree` → `true` (или `да`)
- `priceText` → `Бесплатно`

Пример: `true` в `isFree`, в `priceText` — «Бесплатно».

---

## 8. Как указать пробный урок

- `hasFreeTrial` → `true`
- `isFree` → обычно `false` (программа платная, но есть пробный доступ)

---

## 9. Где вставлять официальную ссылку

Колонка **`officialUrl`** — адрес страницы программы на сайте школы или платформы.

Пример: `https://school.example.com/course/chatgpt`

Оставьте пустым, если официальной ссылки нет. **Не используйте** заглушки вроде `#` или `example.com`.

---

## 10. Где вставлять партнёрскую ссылку

Колонка **`affiliateUrl`** — ваша персональная партнёрская ссылка.

Именно по ней пользователь перейдёт с сайта, когда нажмёт кнопку записи.

Без реальной партнёрской ссылки не ставьте статус `active`.

---

## 11. Как оставить программу черновиком

В колонке **`status`** укажите:

```
draft
```

Черновики видны при локальной разработке (`npm run dev`), но **не публикуются** на готовом сайте.

Рекомендуем импортировать все новые программы сначала как `draft`, проверить на сайте, и только потом менять на `active`.

---

## 12. Как проверить таблицу без добавления файлов

Откройте терминал в папке проекта и выполните:

```bash
npm run import:courses:dry
```

Команда:

- прочитает таблицу;
- проверит все строки;
- покажет, какие файлы **были бы** созданы;
- **не изменит** проект.

Для проверки своей таблицы сначала сохраните её как `data/courses-import.csv`, затем:

```bash
npm run import:courses:dry -- --input data/courses-import.csv
```

Ошибки сохраняются в папку `data/import-reports/`.

---

## 13. Как выполнить реальный импорт

1. Убедитесь, что файл `data/courses-import.csv` заполнен и сохранён в UTF-8.
2. Запустите проверку: `npm run import:courses:dry -- --input data/courses-import.csv`
3. Если ошибок нет, выполните импорт:

```bash
npm run import:courses
```

По умолчанию читается файл `data/courses-import.csv`.

После импорта проверьте сайт локально:

```bash
npm run dev
```

---

## 14. Как исправлять ошибки

После проверки или импорта смотрите сообщения в терминале.

Типичные ошибки:

| Ошибка                         | Что делать                                                      |
| ------------------------------ | --------------------------------------------------------------- |
| Обязательное поле не заполнено | Заполните указанное поле                                        |
| Недопустимая категория         | Используйте одно из шести значений: `neyroseti`, `marketing`, … |
| Некорректный URL               | Укажите полную ссылку `https://...` или оставьте пустым         |
| Файл уже существует            | Измените `slug` или удалите старый файл вручную                 |
| Дублирующийся slug             | В таблице два одинаковых `slug` — сделайте уникальными          |

Подробный отчёт: папка **`data/import-reports/`**.

Исправьте таблицу и снова запустите `npm run import:courses:dry`.

---

## 15. Как активировать проверенный курс

1. Откройте созданный файл в `src/content/courses/ваш-slug.md` **или** измените `status` в CSV на `active` и переимпортируйте с флагом `--force` (только если понимаете, что перезаписываете файл).
2. Убедитесь, что заполнены все обязательные поля и есть `affiliateUrl`.
3. В файле или CSV установите `status: active`.
4. Соберите сайт: `npm run build` — при ошибках валидации сборка подскажет, что исправить.

**Проще всего:** откройте `.md` файл в редакторе и замените `status: draft` на `status: active`.

---

## 16. Как удалить ошибочно импортированный курс

1. Найдите файл в `src/content/courses/` по имени `slug.md`.
2. Удалите этот файл.
3. Уберите строку из CSV или исправьте данные.
4. При необходимости снова запустите импорт.

Демонстрационные файлы `demo-*.md` лучше не трогать — они нужны для предпросмотра шаблона.

---

## Как получить партнёрскую ссылку в ИнфоХит

Общий порядок (интерфейс партнёрского кабинета может отличаться):

1. **Зарегистрируйтесь** в партнёрской программе ИнфоХит (или войдите в существующий аккаунт).
2. **Найдите нужную программу** в каталоге партнёрки.
3. **Активируйте партнёрскую программу** для этой школы или конкретного курса — обычно есть кнопка вроде «Подключить» или «Получить ссылку».
4. **Скопируйте персональную ссылку** — она будет отличаться от обычной ссылки на сайт школы.
5. **Вставьте её в колонку `affiliateUrl`** в вашей CSV-таблице.
6. **Официальную страницу** курса на сайте школы (без партнёрских меток) при необходимости укажите в `officialUrl`.

Не публикуйте программу со статусом `active`, пока не проверили, что партнёрская ссылка открывается и ведёт на нужную страницу.

---

## Защита данных

Система импорта **не перезаписывает** существующие файлы без явного флага `--force`.

Также защищено:

- дублирование `slug` в одной таблице;
- публикация со статусом `active` без обязательных полей и партнёрской ссылки;
- фиктивные ссылки (`#`, `example.com` и подобные);
- перезапись демонстрационных файлов `demo-*.md`.

Новые программы по умолчанию импортируются как **`draft`**.

## Краткая шпаргалка команд

| Команда                                                         | Действие                                     |
| --------------------------------------------------------------- | -------------------------------------------- |
| `npm run import:courses:dry`                                    | Проверка шаблона без изменений               |
| `npm run import:courses:dry -- --input data/courses-import.csv` | Проверка вашей таблицы                       |
| `npm run import:courses`                                        | Реальный импорт из `data/courses-import.csv` |

---

Если что-то непонятно — начните с демонстрационной строки в шаблоне и команды `npm run import:courses:dry`. Так вы увидите, как система обрабатывает данные, без риска для сайта.
