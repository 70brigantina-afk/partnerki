# Навигатор обучения

Каталог онлайн-обучений с партнёрскими программами. Сайт помогает выбрать направление, сравнить программы и перейти на официальный сайт обучения.

## Быстрый старт

### 1. Установка зависимостей

Откройте папку проекта в терминале и выполните:

```bash
npm install
```

### 2. Запуск сайта локально

```bash
npm run dev
```

Сайт откроется по адресу `http://localhost:4321`.

В режиме разработки по умолчанию показываются демонстрационные записи со статусом `draft`.

### 3. Сборка для публикации

```bash
npm run build
```

Готовые файлы появятся в папке `dist/`.

### 4. Просмотр собранной версии

```bash
npm run preview
```

## Проверки

| Команда              | Назначение                           |
| -------------------- | ------------------------------------ |
| `npm run check`      | Проверка TypeScript и Astro          |
| `npm run build`      | Production-сборка + валидация курсов |
| `npm run format`     | Проверка форматирования              |
| `npm run format:fix` | Автоформатирование                   |
| `npm run lint:links` | Базовая проверка внутренних ссылок   |

## Где менять название сайта

Файл: `src/config/site.ts`

Измените поля `name`, `tagline`, `description`, `domain`, `email` и другие настройки. Название автоматически подставится в шапку, подвал и SEO.

## Где менять тексты категорий

Файл: `src/config/categories.ts`

Здесь описания, FAQ, SEO-заголовки и подкатегории для шести направлений.

## Как добавить новое обучение

1. Создайте файл в папке `src/content/courses/`, например `moya-programma.md`.
2. Имя файла станет адресом страницы: `/course/moya-programma/`.
3. Заполните поля в начале файла (frontmatter) по образцу демонстрационных записей.
4. Установите `status: "active"`, когда программа готова к публикации.

Сгенерировать демонстрационные записи заново:

```bash
npx tsx scripts/generate-demo-courses.ts
```

## Как добавить партнёрскую ссылку

В файле программы укажите:

```yaml
affiliateUrl: 'https://example.com/partner-link'
officialUrl: 'https://example.com/official-page'
```

Кнопка «Перейти к обучению» использует `affiliateUrl`. Если поле пустое — кнопка неактивна с текстом «Ссылка будет добавлена после проверки программы».

Все внешние переходы проходят через компонент `src/components/AffiliateButton.astro`.

## Как сделать программу бесплатной или платной

```yaml
isFree: true # бесплатная
isFree: false # платная
hasFreeTrial: true # есть пробный урок
```

## Как вывести программу в рекомендуемые

```yaml
featured: true # блок «Популярные» на главной
recommended: true # блок «Рекомендуемые» на странице категории
priority: 10 # приоритет сортировки (чем выше, тем раньше)
```

## Как временно скрыть программу

```yaml
status: 'draft' # черновик — виден только в dev
status: 'paused' # скрыт с сайта
status: 'archived' # архив
status: 'active' # опубликован
```

## Как изменить дату проверки

```yaml
checkedAt: 2026-07-09
updatedAt: 2026-07-09
```

## Как добавить изображение

Поместите файл в `public/images/courses/` и укажите:

```yaml
image: '/images/courses/nazvanie.webp'
```

Если изображения нет — показывается тематическая заглушка.

## Как подключить Яндекс Метрику

1. Скопируйте `.env.example` в `.env`.
2. Укажите ID счётчика:

```
PUBLIC_YANDEX_METRIKA_ID=12345678
```

3. Пересоберите сайт. Скрипт подключится автоматически, клики по партнёрским ссылкам отправляют цель `affiliate_click`.

## Размещение на GitHub Pages

1. Создайте репозиторий на GitHub.
2. В `.env` или настройках CI укажите:

```
PUBLIC_SITE_URL=https://ваш-логин.github.io/имя-репозитория
PUBLIC_BASE_PATH=/имя-репозитория/
```

3. Соберите проект: `npm run build`.
4. Опубликуйте содержимое папки `dist/` (через GitHub Actions или вручную).

## Подключение собственного домена

1. Укажите домен в `src/config/site.ts` → `domain`.
2. Обновите `PUBLIC_SITE_URL` в `.env`.
3. Настройте DNS и HTTPS у хостинга.

## Проверка внешних ссылок

Перед публикацией проверьте `affiliateUrl` и `officialUrl` вручную в браузере или с помощью внешних сервисов проверки ссылок.

## Замена демонстрационных данных

1. Удалите или переименуйте файлы `demo-*.md` в `src/content/courses/`.
2. Создайте новые записи с `status: "active"` и реальными данными.
3. Уберите `isDemo: true` и обновите `disclaimer`.
4. Для production-сборки убедитесь, что `PUBLIC_SHOW_DRAFTS=false` в `.env`.

## Структура проекта

```
src/
  config/          # Настройки сайта, категории, посадочные страницы
  content/courses/ # Данные программ (Markdown)
  components/      # Переиспользуемые компоненты
  layouts/         # Шаблоны страниц
  pages/           # Маршруты сайта
  styles/          # Глобальные стили
  utils/           # Утилиты (курсы, аналитика, UTM)
scripts/           # Генерация демо-данных и проверки
public/            # Статические файлы
```
